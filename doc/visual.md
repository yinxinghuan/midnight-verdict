# 《午夜验客》视觉圣经

> 最终方向：C「午夜怪谈漫画」为基础；B「失真监控录像」仅用于夜客身份揭晓时的一帧横向信号撕裂。A/B/C 原型比较见 `doc/direction-comparison.md`。

## 1. Visual thesis

- 游戏与受众：面向移动端短局玩家的凌晨便利店观察判断游戏，兼具怪谈悬疑与荒诞冷幽默。
- 情绪承诺：熟悉的日常像一页廉价怪谈漫画逐格失控，玩家通过明确线索破解异常，并在夸张揭晓中释放紧张。
- 一句话视觉命题：一间被粗墨线、网点和错版油墨印在旧纸上的午夜便利店，收银设备、角色与 UI 都属于同一本怪谈漫画。
- 标志性视觉时刻：朱红印章落下，画面被一条来自监控录像的横向信号撕裂切开；夜客黑色影子突破漫画格，酸黄色冲击线指向决定性线索。
- 三个必须品质：粗粝但清楚、诡异但不血腥、夸张但有节制。
- 三个避免方向：写实影视截图常态化、持续监控滤镜、日式动漫与通用超级英雄漫画。

## 2. Composition and camera

- 方向与比例：竖屏响应式 DOM 舞台，基准 390×844；覆盖 320×568 至 430×932，桌面最大舞台宽 430 px。
- 镜头：收银台后的平视固定镜头，等效 50 mm；顾客腰部以上构图，不使用 B 方向的常态鱼眼高机位。
- 游戏焦点：顾客面部、手中商品和柜台反射组成三角线索区，占安全画面中上部 58%。
- 前景：粗墨线柜台、收银数字、实体判断键。
- 中景：顾客、商品、对白格和三个酸黄色线索热点。
- 背景：货架、冷柜与顶灯，以低密度网点和深紫黑块退后。
- HUD 安全区：顶部避让 `safe-area-inset-top`；底部判断区避让 `safe-area-inset-bottom`，操作下缘距安全区至少 12 px。
- 注意力路径：角色眼神 → 漫画对白 → 酸黄热点 → 黄/朱红判断键；揭晓时黑影与信号撕裂短暂打破路径。
- 开始页注意力路径：海报中的黄色雨衣主角 → 周围黑影 → HUMAN / NIGHT GUEST 前景牌 → 底部朱红“开始值班”按钮；不再使用空货架作为主视觉。

## 3. Color

- 墨黑紫 `#1D1022`：轮廓、阴影、桌面外部背景。
- 深茄紫 `#3E1747`：店内暗面、控制区与夜色。
- 旧纸奶油 `#F1E5C5`：对白、结果、HUD 纸面。
- 酸性黄 `#F3F02D`：线索热点、人类判断键、冲击线和高价值提示。
- 印刷朱红 `#ED4A37`：夜客判断键、误判印章与稀有强调。
- 柜台绿 `#2E5B45`：便利店场景锚点，避免整页只有紫黄红。
- 冷柜青 `#65B9AE`：少量环境反光，不作为 UI 主强调。
- 正确绿 `#79A96B`：结果印章边码；正确/错误同时依赖文字与形状。
- 使用比例：墨黑紫与深茄紫 45%，旧纸奶油 28%，柜台绿 15%，酸黄 7%，朱红 5%。
- 禁止：紫蓝霓虹渐变、发光青红对撞、大面积纯黄背景、常态绿色监控单色滤镜。

## 4. Typography

- 标题：Onest 850，中文使用系统黑体粗字回退；字距 `-0.04em`，可配 2–3 px 墨黑硬偏移，不使用发光。
- UI / 正文：Onest 550/750，中文回退 `"Noto Sans SC", "PingFang SC", "Microsoft YaHei", sans-serif`。
- 数字 / HUD：JetBrains Mono 500，用于时间、分数、序号和小票编号。
- 移动端标题 42–52 px；屏幕标题 24–30 px；正文 16–18 px；HUD 11–14 px；按钮 18–20 px。
- 对白允许粗体关键词，但不使用全篇漫画拟声字体；英文标签可大写并使用 `0.06em` 字距。
- 正文不描边；按钮与评级仅用硬偏移阴影增强层级。

## 5. Shape, material, and lighting

- 主导形状：直角漫画格、3–5 px 粗黑边、轻微错位矩形与圆形线索热点；常规圆角不超过 5 px。
- 描边：场景和主要面板 4–5 px；按钮 3 px；自绘 SVG 图标保持 1.8 px 圆端线，但置于粗边容器中。
- 阴影：使用 3–7 px 纯墨黑硬阴影；禁止玻璃模糊和柔亮外发光。
- 材质：旧纸、粗网点、丝网印刷错版、干刷墨线；纹理不得降低 16 px 正文可读性。
- 光线：角色仍保持上方偏左主光与右后冷柜轮廓光，但翻译为奶油高光、深紫阴影和少量青色边缘。

## 6. Characters, environments, and assets

- 人物比例：现实成人比例的半身漫画角色，轮廓先于五官细节可读；年龄、体型和穿着多样但不作为身份线索。
- 线条：统一粗干刷外轮廓、较细面部线、固定网点密度；皮肤只保留 2–3 个明暗层级。
- 表情：常态克制疲惫；人类揭晓为尴尬或松口气；夜客为冷静、荒诞或干燥幽默，不做血腥身体恐怖。
- 相机与裁切：同一便利店平视镜位，人物腰部以上，头顶完整；关键商品和至少一个线索必须可见。
- 导出：原始 1024×1024 JPEG/PNG，同一漫画参考图做 `ref_url`；显示不低于 640 px 高。
- 首版内容：12 位正常态顾客，5 位夜客各有一张揭晓态；背景可随角色图保留，但镜位、柜台高度和冷柜位置要一致。
- 生成门禁：逐张检查生成文字、异常手指、血迹、动漫化、写实回退、镜位漂移和网点密度。

## 7. UI and icons

- 图标：自绘便利店线性 SVG 家族，24×24 viewBox、1.8 px 圆端线；暂停、声音、关闭、放大镜、经理电话与返回一致。
- 主判断键：人类为酸黄配墨黑，夜客为朱红配墨黑，3 px 粗边与 4 px 硬阴影；面积相同。
- 按下态：下移 3 px、硬阴影减至 1 px，80 ms 完成。
- 聚焦态：3 px 酸黄或旧纸外框加 2 px 间距，不只靠颜色。
- 禁用态：网点覆盖、透明度 0.55，并显示明确文字。
- HUD：像贴在画面顶部的三格漫画说明框，旧纸底、粗墨边；常态安静，最后 5 秒才切朱红边。
- 对白：旧纸横向格，使用粗墨上下边与轻网点，不做通用圆角气泡。
- 当班规则：场景左上使用一张深紫标题价签和两张酸黄短标签，只显示 4–7 字关键词；完整解释通过标题属性和班前守则表达，不遮挡角色面部。
- 连判奖励：揭晓小票内部使用酸黄与旧纸两张并列撕票，保持 48 px 高；不可恢复经理时使用网点覆盖、降透明度与原因文字共同表达禁用。
- 排行榜：冠军入口是粗边酸黄横条，展示圆形头像/首字母、冠军名和分数；完整榜单沿用旧纸网点、深紫粗边、朱红名次章，不使用通用玻璃拟态或 Emoji 奖牌。
- 加载：漫画格逐格显影，不使用旋转圆环。
- 开屏海报：直接复用正式 `poster.png`，保持其叙事构图，不把 UI 截图或程序化货架当主视觉；海报用 4 px 墨边嵌入舞台，上方不再叠加额外大标题。
- Emoji 政策：功能图标、HUD、状态与操作说明不得使用 Emoji。

## 8. Motion and VFX

- 性格：像印刷格被快速推入画面；设备按压短促，纸张与漫画格稍柔，破格只用于揭晓。
- token：按压 80 ms；热点 140 ms；线索格 180 ms；顾客入场 360 ms；结果 1,100–1,500 ms。
- 换客：旧角色随场景节点立即卸载，深紫卷帘从上下合拢覆盖解码窗口；新图就绪后以 180 ms 上下分离，不做旧图与新图的透明交叉淡化。
- 正确：8–12 个小纸屑/网点碎片，奶油与酸黄两色，寿命 360–520 ms。
- 连判：HUD 计数以一格错版位移闪动 160 ms，不用全屏 Combo 字。
- 夜客揭晓：印章接触后出现 70 ms 横向信号撕裂（来自 B）；随后黑影破格 260 ms、酸黄冲击线 180 ms、结果格送入 240 ms。
- 误判：朱红错印偏移 3 px，画面墨版错位 160 ms；禁止持续抖屏和反复闪烁。
- 屏幕震动：只在第 3 次误判时允许 1 px、2 帧；普通判断无震动。
- 减弱动态：取消信号撕裂、破格位移、粒子和震动，以静态黑影、朱红/绿色印章及原因文字替代。

## 9. References translated into principles

- 地下怪谈漫画：采用粗墨、有限色盘和日常环境中的不可能阴影；不复制现成怪物或超级英雄构图。
- 廉价丝网印刷：采用错版边缘、网点和硬色块；不模拟影响正文的脏污与破损。
- 单场景舞台：固定镜头让 12 位顾客成为内容变化；不频繁切换背景和透视。
- B 方向监控录像：仅借用一次横向 tracking tear 表达现实信号被异常打断；不保留扫描线、鱼眼和时间码作为常态 UI。

## 10. Anti-patterns

- 禁止 Emoji、混搭彩色扁平图标、霓虹赛博图标和写实照片按钮。
- 禁止常态 CRT 扫描线、鱼眼黑边、绿色夜视、玻璃拟态和紫蓝发光渐变。
- 禁止每次判断都破格爆炸；普通人类与低强度反馈必须克制，才能让夜客揭晓升级。
- 禁止人物线宽、网点、镜头高度、柜台比例和有限色盘不一致。
- 禁止血迹、伤口、病容或现实身份特征成为“夜客”暗示。
- 禁止自动生成的伪文字成为真实线索；所有关键线索由 DOM 文本明确表达。
- 漂移示例：一位角色突然变真人照片、按钮使用圆润 App 卡片、每个背景换透视、用红色喷溅替代印章。

## 11. Final acceptance and QA

- 开始页：即使不读标题，也能从粗墨便利店、旧纸守则、酸黄/朱红判断键识别怪谈漫画世界。
- 游戏页：320×568 下顾客、对白、3 个热点、倒计时和判断键完整可达；线索热点命中实际物体。
- 高反馈：夜客揭晓包含一次短促 B 信号撕裂，随后回到 C 的黑影破格与结果格；静音/减弱动态仍能理解。
- 结算：评级是唯一主角，统计像漫画末页收银小票，重新值班是唯一主按钮。
- 窄屏：中文与长英文均不溢出；不依赖整页 transform 缩小。
- QA 结果（2026-07-14）：生产构建已覆盖 320×568、390×844、430×932、1280×800，以及 320×568 英文界面；各尺寸无横向溢出，所有按钮命中区均不小于 44×44 px。
- v1.1 QA（2026-07-15）：复验开始页冠军入口、站外榜单 CTA、三行模拟平台榜单、长用户名省略、本人行禁用、他人主页跳转、证据规则、第三次深查、连判奖励和 S 级结算；320×568 无横向溢出，交互目标均不小于 44×44 px，控制台错误为 0。
- 开屏重做 QA（2026-07-15）：390×844 与 320×568 均完整呈现海报主角、黑影、双判断牌、本地标题、开始按钮和排行榜入口；无横向溢出或小于 44 px 的按钮。人工延迟第二位顾客图片 650 ms 时，上一位图片节点已卸载，卷帘保持关闭；新图完成后再打开，控制台错误为 0。
- 流程结果：8 人全对班次、3 次误判提前结束、经理代判一次性限制、后台暂停恢复、快速重复提交均通过自动化验证。
- 素材结果：12 张常态角色、5 张夜客揭晓图与 1024×1024 正式海报逐张检查；海报同时通过 160×160 缩略图检查，未使用 UI 截图或程序化图形。
- 最终缺陷：P0 0、P1 0；无阻断交付的 P2。
