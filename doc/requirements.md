# 《午夜验客》需求文档

## 1. Overview

《午夜验客》是一款单局 3–5 分钟的竖屏观察判断游戏：玩家扮演凌晨便利店店员，通过检查顾客、商品与对话中的超自然破绽，判断柜台前的客人究竟是普通人类还是伪装成人类的“夜客”。

### 玩家与体验目标

- 目标玩家：喜欢短篇怪谈、黑色幽默和轻量推理的移动端玩家。
- 使用场景：手机单手游玩，通勤、排队或睡前完成一局。
- 核心交互：查看线索，在倒计时结束前按下“人类”或“夜客”印章。
- 情绪承诺：先感到不安和怀疑，再因荒诞线索与揭晓反差发笑。
- 内容边界：判断依据只能是虚构的超自然现象，不使用种族、国籍、性别、年龄、残障、穿着、职业或其他现实身份特征作为危险暗示。

### 屏幕与状态

1. 开始页：正式叙事海报作为第一视觉，必须同时看见黄色雨衣主角、包围黑影与 HUMAN / NIGHT GUEST 冲突；下方值班控制台承载本地化标题、开始按钮、最佳分与当班冠军入口。
2. 班前简报：用 3 张短卡说明查看线索、作出判断和误判后果。
3. 营业中：顾客主体、柜台、对话、当班抽取的 2 条可靠证据规则、3 个可检查线索点、剩余时间、失误数、连判数和两个判断按钮。
4. 线索特写：点击线索后在原场景上弹出局部特写和一句观察描述，不离开主流程。
5. 判定反馈：人类直接展示结果小票；夜客先完整停留在异常揭晓图，由玩家主动进入判决说明，再展示关键破绽、得分变化与顾客的荒诞反应。
6. 暂停页：继续、重新开始、声音开关和退出本局。
7. 连胜奖励：连续正确第 3、6 位顾客后，在揭晓单上选择“下位顾客 +3 秒”或“恢复经理求助”。
8. 结算页：当班评级、正确数、连续正确最高值、总分、最佳分、冠军入口和重新值班按钮。
9. 排行榜：展示名次、头像、名字、分数与本人标识；其他玩家行可打开其 AlterU 主页，站外环境显示下载 AlterU 入口。

## 2. Visual Design

- 目标为“凌晨便利店的熟悉感被一点点扭歪”：柜台、微波炉、收银机、冷柜灯和廉价商品都应可信，但每位夜客至少带有一个不可能发生的视觉细节。
- 核心画面为固定竖屏柜台舞台，顾客占上方约 55%，对话和线索位于中段，判断操作固定在底部安全区上方。
- UI 不是独立悬浮卡片集合，而应像便利店本身的设备与耗材：小票、价签、收银机显示、橡皮印章和塑料按钮。
- 顾客需要一组风格统一的粗墨网点漫画半身角色图，并为夜客身份揭晓准备黑影破格或独立反应状态；商品、影子、小票和反射需要可独立检查的细节素材。
- 功能图标统一使用同一套自绘 SVG 线性图标，不使用 Emoji 作为按钮、HUD 或操作说明图标。
- 视觉方向、颜色、字体、材质、光线和动效将在 `doc/visual.md` 中于方向确认后确定。

## 3. Game Mechanics

### 单局结构

- 每局固定接待 8 位顾客。
- 顾客身份配比为 5 位人类与 3 位夜客，每局洗牌；首位顾客固定为教学用人类。
- 玩家初始有 3 次容错；误判一次失去 1 次容错，容错归零时当班立即结束。
- 每位顾客有 20 秒判断时间。倒计时前 2 秒只做入场演出，18 秒时开放操作。
- 每位顾客提供 3 个线索点，其中至少 1 个与真实身份直接相关；其余可以是中性信息或喜剧干扰项。
- 每个线索点每位顾客只能检查一次，打开特写时倒计时暂停，关闭后继续。
- 每班从 4 条“可靠证据规则”中随机抽取 2 条，规则分别强调镜像/影子、票据/日期、温度/物理、声音/时序；规则是观察优先级提示，不改变顾客真实身份与计分上限。
- 每位顾客前 2 个首次检查免费；第 3 个不同线索属于深度检查，在打开特写前立即扣除 2 秒，最低扣至 0.1 秒。重复打开已检查线索不再扣时。
- 每局有 1 次“呼叫夜班经理”机会：自动给出正确身份并继续流程，但该顾客不计分且中断连判。
- 连续正确达到第 3、6 次时触发一次奖励选择；“加时”让下一位顾客由 18 秒变为 21 秒，但时间奖励仍以最多 18 秒计算；“恢复经理”仅在经理已经使用后可选。奖励必须选择后才能进入下一位顾客。

### 计分与连判

- 正确判断基础分为 100 分。
- 时间奖励为 `floor(剩余秒数 × 5)`，范围 0–90 分。
- 连判奖励从连续正确第 2 次开始增加，每次为 `min((连续正确数 - 1) × 25, 100)` 分。
- 检查全部 3 个线索不会扣分；鼓励观察，不把快速盲猜设为唯一最优解。
- 误判得 0 分、失去 1 次容错并清空连判；超时按误判处理。
- 理论单局常规最高分为 2,070 分：8 次正确基础分 800、满时间奖励 720、连判奖励 550。
- 排行榜提交每局最终总分，只保留平台返回的个人最好成绩；规则抽取不提供额外分，确保不同班次可比较。

### 判定与反馈节奏

- 按下判断按钮后 80 毫秒内显示按钮压下和印章落点。
- 220 毫秒时冻结倒计时并展示身份揭晓。
- 夜客身份揭晓后进入双阶段反馈：第一阶段只展示完整异常图与底部控制带，不自动推进且不以大面积遮罩覆盖顾客；玩家点击“查看判决说明”后，第二阶段才送入结果小票。
- 人类身份保持单阶段反馈，220 毫秒揭晓后直接显示结果小票，不增加一次确认操作。
- 正确判定反馈持续 1.1 秒；误判反馈持续 1.5 秒。
- 判定结束后用 450 毫秒切换到下一位顾客。
- 正确时产生 10–14 个短促纸屑或收银数字粒子；误判时灯光闪烁 2 次，总时长 320 毫秒。

### 内容规模

- 首个可发布版本包含 12 位顾客剧本：7 位人类、5 位夜客。
- 每位顾客包含姓名或代号、购买物、开场对白、3 个线索、身份揭晓文案和正确/错误反应。
- 同一局不重复顾客；连续两局尽量避免出现完全相同的 8 人组合。
- 开始页与营业期间仅预取“当前 + 下一位顾客”的常态图与揭晓图，不批量解码全部 17 张大图；进入下一位时必须卸载上一位图片节点，并用卷帘交接遮罩覆盖新图解码窗口。卷帘由 `load/error`、DOM `complete` 或 `decode()` 任一路解除，并设 1.2 秒保险上限，禁止永久停留在加载状态。
- 夜客线索只使用超自然或逻辑不可能事件，例如镜中缺席、影子方向错误、商品温度违反常识或小票日期来自未来。

## 4. Controls

### Pointer / Touch

- 点击顾客或柜台上的线索热点打开线索特写；热点触控范围至少 44×44 CSS px。
- 点击特写外区域或关闭按钮返回营业画面。
- 连胜奖励的两个选项为揭晓单内普通按钮；“恢复经理”不可用时显示禁用状态和原因。
- 夜客异常图阶段的“查看判决说明”使用 `onClick`；异常图停留时长完全由玩家决定。
- “人类”“夜客”和“呼叫经理”使用 `onPointerDown`，按下即提供视觉反馈，并在同一次指针操作中锁定，防止重复提交。
- 暂停、声音、特写关闭等普通 UI 控件使用 `onClick`。
- 排行榜入口与关闭按钮使用 `onClick`；可滚动榜单中的其他玩家行使用 `onClick`，滑动列表不会误开主页。

### Keyboard

- `1`、`2`、`3`：检查对应线索。
- `H`：判定为人类；`N`：判定为夜客。
- `M`：呼叫夜班经理。
- `Escape`：关闭线索特写；无特写时打开或关闭暂停页。
- 所有键盘操作只在游戏画面获得焦点且未处于判定锁定状态时生效。

### 平台与适配

- 主要平台为 Aigram 内嵌移动 Web，竖屏优先。
- 设计基准为 390×844 CSS px；必须覆盖 320×568、375×667、390×844、430×932 与 1280×800。
- 底部操作区避让 `env(safe-area-inset-bottom)`，顶部 HUD 避让 `env(safe-area-inset-top)`。
- 支持中文与英文；所有用户可见文字通过轻量 `t()` 系统输出，并测试英文长文案换行。
- 支持 `prefers-reduced-motion`；减弱模式保留判定结果但取消闪烁、抖动和大幅位移。

## 5. Win / Lose Conditions

### 胜利

- 在容错归零前完成 8 位顾客即视为完成当班。
- 评级按正确数计算：8 位正确为 S，7 位为 A，6 位为 B，4–5 位为 C，0–3 位为 D。
- 使用“呼叫夜班经理”的顾客不算错误，但最高评级降一级，S 最低降为 A。

### 失败

- 第 3 次误判或超时时当班立即结束，剩余顾客不再出现。
- 失败结算必须明确展示导致下班的最后一位顾客、被忽略的关键线索和可重试入口。

### 结算内容

- 当班评级、总分、历史最佳分。
- 正确判断数 / 已接待人数、最高连续正确数、剩余容错。
- 一条依据表现变化的夜班主管评语。
- 当班冠军头像、名字与分数入口；榜单为空时显示“排行榜”。
- “重新值班”为唯一主按钮；声音和返回入口为次级操作。

## 6. Sound Effects

首版使用 Web Audio 合成短音效，不依赖外部音频文件；首次玩家交互后创建或恢复 AudioContext。

- 线索打开：正弦波 620→760 Hz，80 毫秒，快速起音 5 毫秒、释放 55 毫秒，音量 0.10。
- 线索关闭：正弦波 520→420 Hz，70 毫秒，音量 0.07。
- 按钮压下：三角波 180 Hz，45 毫秒，音量 0.08。
- 印章落下：白噪声 55 毫秒叠加 95 Hz 正弦波 90 毫秒，音量 0.16。
- 正确判定：三音上行 440 / 554 / 659 Hz，每音 70 毫秒、间隔 25 毫秒，三角波，音量 0.12。
- 连判 3 次及以上：在正确音末尾追加 880 Hz 正弦波 100 毫秒，音量随连判从 0.08 增至 0.14。
- 深度检查扣时：方波 310→220 Hz，90 毫秒，音量 0.07，并让计时数字错版闪动 120 毫秒。
- 连胜奖励出现：三角波 523 / 659 / 784 Hz，各 65 毫秒；选择加时或经理恢复时追加 988 Hz 正弦波 90 毫秒，音量 0.10。
- 误判：锯齿波 150→72 Hz，280 毫秒，起音 10 毫秒、释放 160 毫秒，音量 0.13。
- 倒计时最后 5 秒：每秒一次 880 Hz 方波脉冲，45 毫秒，音量 0.06；最后 2 秒升至 1,100 Hz。
- 夜客揭晓：60 Hz 正弦波 350 毫秒叠加经低通处理的噪声 240 毫秒，音量 0.12。
- 静音状态保存到 `localStorage`；页面进入后台时暂停倒计时并停止所有持续声音。
